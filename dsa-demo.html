Conversation Graph:
Inferring Communication Roles from Sentiment-Weighted Interactions

A custom doubly-linked-list sliding temporal window pairs with a directed weighted graph to model team communication from a transcript. Each utterance is scored with VADER, edges inherit the average sentiment of the interactions they represent, and each speaker's role is inferred from in- and out-degree. The original notebook code runs unmodified in your browser — Python compiled to WebAssembly via Pyodide.

How it works

01 · WINDOW

Sliding Temporal Window

A doubly-linked list keeps every utterance from the last w seconds. As the transcript advances, old events are popped from the head in O(1) and new ones appended at the tail. The list is the conversational "short-term memory" — anyone still in it is considered an addressable peer.

02 · GRAPH

Directed Weighted Graph

Each new utterance creates a directed edge from the speaker to every other speaker still inside the window. Repeat interactions accumulate weight on the edge. Self-loops are dropped — speaking doesn't count as addressing yourself.

03 · SENTIMENT

VADER Compound Score

Every utterance is fed through VADER (Valence Aware Dictionary for sEntiment Reasoning) — the same analyzer used in the notebook. Each edge stores the running average of its underlying interactions; that average drives the edge color in the visualization.

04 · ROLES

Role Classification

Once the graph is built, the group's average in-degree and out-degree are computed. Each speaker is then classified relative to those averages: Leader / Coordinator (high in & out), Active Speaker (high out, low in), Frequently Addressed (high in, low out), or Observer / Quiet Participant. Adapts to the conversation rather than using fixed thresholds.

The Python classes (DLLNode, TemporalWindowList, InteractionGraph) and the main processing loop are taken verbatim from the project notebook. They run under Pyodide — a CPython interpreter compiled to WebAssembly — using the official vaderSentiment package (same algorithm and lexicon as NLTK's VADER). Visualization uses vis-network, the same library pyvis wraps under the hood. No algorithmic differences from the notebook — this page is the actual final-project code, executing in your browser.
